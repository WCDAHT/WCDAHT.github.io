How ENF Analysis Works

Electrical Network Frequency (ENF) is the frequency at which AC power is delivered (50Hz or 60Hz). Due to load variations on the power grid, this frequency fluctuates slightly over time. These fluctuations are captured in audio/video recordings through electrical interference.

Forensic Applications:

  • Timestamp verification - Match ENF traces against grid databases
  • Authentication - Detect edited or manipulated recordings
  • Speed detection - Identify if media was sped up or slowed down
  • Geographic location - 50Hz vs 60Hz indicates recording region

Usage Examples

Basic Usage

# Extract ENF from audio file (60Hz region) python main_v3.py recording.wav # Extract ENF from 50Hz region python main_v3.py recording.mp3 -f 50 # Process video file python main_v3.py video.mp4

Advanced Options

# Use GPU acceleration python main_v3.py large_file.wav --use-gpu # Custom analysis parameters python main_v3.py audio.wav \ --window 4.0 \ --overlap 0.75 \ --harmonics 1 2 3 4 # Generate visualization python main_v3.py recording.wav \ --plot \ --plot-output analysis.png

Output Format

# CSV output contains: # Time(s), ENF(Hz), Confidence 0.0, 59.9597, 0.95 1.0, 59.9601, 0.94 2.0, 59.9598, 0.96 ... # Statistics displayed: Mean ENF: 59.9598 Hz Std ENF: 0.0012 Hz Min/Max: 59.957/59.963 Hz

Command Line Options:

  • -f, --frequency: Nominal mains frequency (default: 60.0)
  • -t, --tolerance: Frequency search tolerance in Hz (default: 0.1)
  • -w, --window: Analysis window length in seconds (default: 2.0)
  • -r, --overlap: Window overlap ratio 0-1 (default: 0.5)
  • --sample-rate: Audio sample rate (default: 22050)
  • --harmonics: Harmonics to analyze (default: 1 2 3)
  • --use-gpu: Enable GPU acceleration
  • --plot: Display ENF trace plot
